Introduction

Communication applications are everywhere, whether your tweeting, snapchatting or even meerkatting these tools all require you to be connected to the person you're following/friends-with/subscribed-to.

But what if you were instead connected by your physical location. Localised information becomes more relevant and communities, particularly in dense areas such as a university grounds, would form organically.

In this guide we're going to learn how to build such an application using AngularJS and Pusher, by creating location specific channels that utilise client events. Most of the work will be on the client-side leaving only the authentication layer to build in Express.js.

Here's an outline of the flow -

  • The user lands on the Application and we request their GeoLocation, with a friendly notice to let them know the status.

  • Once we have their location we generate a channel name based on a low precision lookup of their location.

  • Using the Pusher Angular library we subscribe to the private location channel, authenticating the user in doing so.

  • We listen for message events on the channel and push them to a full screen map near the location they occurred.
